# Requirements Checklist: Rulebook Page

## Functional Requirements

- [ ] FR-001: Dedicated `/rules` page with 4 tabs
- [ ] FR-002: Quick-access "?" button in game rooms opens modal
- [ ] FR-003: Content structure (emoji, name, 1-2 sentence description)
- [ ] FR-004: Roles tab with all 8 roles by team
- [ ] FR-005: Game Modes tab (Lady, Decoy, Split Intel)
- [ ] FR-006: Visual Guide tab (indicators, colors, symbols)
- [ ] FR-007: Game Flow tab (phases, win conditions)
- [ ] FR-008: Responsive design (mobile + desktop)
- [ ] FR-009: Navigation links (home footer, game room header)

## Non-Functional Requirements

- [ ] NFR-001: <1 second load time (static content)
- [ ] NFR-002: Keyboard accessible tabs
- [ ] NFR-003: Consistent with existing design system

## User Stories

- [ ] US1: New player can learn rules before joining game
- [ ] US2: In-game player can reference symbols/colors
- [ ] US3: Player can understand their role's abilities

## Success Criteria

- [ ] `/rules` page renders with 4 tabs
- [ ] All 8 roles displayed with correct team colors
- [ ] All 3 game modes documented
- [ ] Visual guide covers all UI indicators
- [ ] Game flow explains phases and win conditions
- [ ] "?" button appears in game room
- [ ] Modal opens with same content as page
- [ ] Works on mobile (375px) and desktop
- [ ] Matches existing dark theme aesthetic

## Design Decisions Recorded

- [x] Access: Both dedicated page + quick-access button
- [x] Organization: Tabbed sections
- [x] Quick-access behavior: Full modal (same as page)
- [x] Content depth: Concise with visual emphasis
- [x] Tab structure: 4 tabs (Roles / Game Modes / Visual Guide / Game Flow)
