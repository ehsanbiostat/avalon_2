# Requirements Checklist: Rulebook Page

## Functional Requirements

- [x] FR-001: Dedicated `/rules` page with 4 tabs
- [x] FR-002: Quick-access "?" button in game rooms opens modal
- [x] FR-003: Content structure (emoji, name, 1-2 sentence description)
- [x] FR-004: Roles tab with all 8 roles by team
- [x] FR-005: Game Modes tab (Lady, Decoy, Split Intel)
- [x] FR-006: Visual Guide tab (indicators, colors, symbols)
- [x] FR-007: Game Flow tab (phases, win conditions)
- [x] FR-008: Responsive design (mobile + desktop)
- [x] FR-009: Navigation links (home footer, game room header)

## Non-Functional Requirements

- [x] NFR-001: <1 second load time (static content)
- [x] NFR-002: Keyboard accessible tabs
- [x] NFR-003: Consistent with existing design system

## User Stories

- [x] US1: New player can learn rules before joining game
- [x] US2: In-game player can reference symbols/colors
- [x] US3: Player can understand their role's abilities

## Success Criteria

- [x] `/rules` page renders with 4 tabs
- [x] All 8 roles displayed with correct team colors
- [x] All 3 game modes documented
- [x] Visual guide covers all UI indicators
- [x] Game flow explains phases and win conditions
- [x] "?" button appears in game room
- [x] Modal opens with same content as page
- [x] Works on mobile (375px) and desktop
- [x] Matches existing dark theme aesthetic

## Design Decisions Recorded

- [x] Access: Both dedicated page + quick-access button
- [x] Organization: Tabbed sections
- [x] Quick-access behavior: Full modal (same as page)
- [x] Content depth: Concise with visual emphasis
- [x] Tab structure: 4 tabs (Roles / Game Modes / Visual Guide / Game Flow)
